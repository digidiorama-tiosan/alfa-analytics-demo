Production Blueprint · Engineering Handoff

From a clickable prototype to a governed, multi-user product.

The interface and behavior are already built and validated. This blueprint specifies the backend that makes it real — shared across users and devices, secure, and able to scale past the browser.

The two HIGH-priority constraints — and how we close them
HIGH · Governance

Single-user, no governance

No auth, RBAC or audit. → Real sign-in, self-service registration with admin approval, role-based permissions enforced on the server, and an immutable audit trail.

HIGH · Scale

Browser memory ceiling

All rows live in the browser. → Data moves server-side; aggregations run in SQL/warehouse and the browser receives KB of rolled-up JSON — admin uploads once, everyone sees it.

Recommended stack — two pragmatic paths
Fastest

Supabase (managed)

  • Auth — email/password + JWT, MFA-ready
  • Postgres — use the provided schema as-is
  • Storage — uploaded files
  • Realtime — free data_updated push
Full control

Node + Postgres (self-hosted)

  • Express API — the runnable backend-starter/
  • JWT + bcrypt/argon2 auth
  • S3 / MinIO object storage
  • SSE / ws realtime · on-prem friendly
